# 游戏状态保存与加载功能

## 功能概述

本游戏实现了完整的状态保存与加载功能，允许玩家在任何时候暂停游戏并将当前进度保存下来，之后可以从保存的状态继续游戏。

## 技术实现

### GameStateManager.ts

核心的状态管理器，负责：

1. **数据序列化**：捕获当前游戏状态并转换为JSON格式
2. **本地存储**：使用浏览器的localStorage保存游戏数据
3. **状态恢复**：从保存的数据重建游戏状态
4. **数据验证**：确保保存和加载过程的安全性

### 保存的数据包括：

- 游戏得分
- 当前等级
- 已消除的层数
- 游戏结束状态
- 网格系统状态（已放置方块的位置）
- 当前方块的位置和形状
- 游戏内部计时器状态

## 使用方法

### 键盘快捷键

- **S键**：保存当前游戏状态
- **L键**：加载最近保存的游戏状态
- **P键**：暂停/继续游戏

### 自动处理

- 保存游戏时会自动暂停游戏，保存完成后恢复
- 加载游戏时会暂停当前游戏，加载完成后恢复
- 如果加载失败，会自动恢复游戏

## 状态管理接口

除了键盘快捷键，还可以通过代码调用：

```typescript
// 保存游戏
const gameStateManager = // 获取GameStateManager组件
gameStateManager.saveGame();

// 加载游戏
gameStateManager.loadGame();

// 检查是否有保存的游戏
const hasSave = gameStateManager.hasSavedGame();

// 删除保存
gameStateManager.deleteSave();
```

## 数据安全

- 所有游戏数据均保存在用户的本地浏览器中
- 不涉及任何网络传输
- 用户可以随时删除保存的数据
- 数据仅包含游戏状态，不含个人身份信息

## 注意事项

1. 保存的游戏数据仅在当前设备和浏览器中有效
2. 清除浏览器数据会导致游戏存档丢失
3. 在不支持localStorage的环境中无法使用此功能
4. 建议在游戏进行中定期保存，以防意外中断

## 与现有系统的集成

- 与游戏暂停机制完美结合
- 不影响原有的游戏逻辑
- 支持游戏控制器的全部功能
- 与网格系统和方块组件无缝协作
